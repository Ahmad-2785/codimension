Non-Coding Contribution

Introduction

The Codimension project is not limited to coding. It is an interactive tool so many decisions of how to interact with the users are made during the development. These decisions may well be not the best, and this is the area for the improvements too.

The project also uses a generated graphics for flowcharts. The design of the elements was suggested by non-professional designers and it is believed it could also be improved.

Your Ideas

If you have ideas of what could be done better, please submit an issue in the Codimension IDE bug tracking system on github having an appropriate label set for it. The suggested label is 'enhancement'.

UI Design

If you can draw better pixmaps for the UI buttons, suggest better graphics for the flowchart diagrams, suggest better layout of the dialogs or a better workflow, then please let us know. You can create an issue in the Codimension IDE bug tracking system on github having an appropriate label set for it. The suggested label is 'enhancement'.

To try your own pixmaps for Codimension you can do the following:

  • Make Codimension running from a git clone as described here
  • Replace a pixmap in ~/codimension/src/pixmaps
  • Restart Codimension to see the new pixmap

Submitting a Bug

This is a valuable information for us so if you spot a bug please submit an issue.

The existing Codimension codebase is split between tree repositories and each of them has its own bug tracker:

  • Codimension IDE bug tracker
  • Brief Python parser (cdm-pythonparser) bug tracker
  • Control flow Python parser (cdm-flowparser) bug tracker

If you are not sure where the bug is, please use the Codimension IDE bug tracker.

Contacts
